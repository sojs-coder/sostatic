Protocol

The same API Apple uses in Xcode

01

One-time setup

The user enables Developer Mode once (Settings → Privacy & Security → Developer Mode) and taps Trust on first USB connect. Both settings persist across reboots and iOS updates — no repeat prompts.

02

USB detection

On each subsequent connect, usbmuxd detects the phone and authenticates with the stored pairing record automatically. Session starts in under 15 seconds.

03

DVT tunnel

The ghostd daemon opens an encrypted RemoteXPC/QUIC tunnel to iOS's developer services via pymobiledevice3 — the same protocol Xcode uses. iOS version is auto-detected; iOS 16–18.x all supported.

04

Location injection

DVT simulate-location feeds coordinates into locationd at 1 Hz with Gaussian jitter (σ=1.5m), variable accuracy, and smooth altitude variation — statistically indistinguishable from real GPS.

Compliance

Required filings
before commercial sale

Required — Pre-launch

FCC Part 15B

Unintentional radiator certification required for US market. The device contains no intentional RF transmitter (Wi-Fi and BLE are disabled). Class B testing at accredited lab. Estimated cost: $5–10k. Self-declaration path available for Class B digital devices.

Required — EU launch

CE / RED Declaration

EU market requires CE marking and Radio Equipment Directive compliance. EMC testing (EN 55032, EN 55035) plus safety (EN 62368-1). Can be combined with FCC testing at same lab. Estimated €8–15k. Required before any EU sale or distribution.

Required — All markets

RoHS / REACH

Material compliance required for EU and most international markets. Supplier declarations for all components; PCB and battery must meet restricted substance limits. Generally handled via supply chain declarations; minimal direct cost.

Not Required

Apple MFi

MFi certification is explicitly not required. The DVT simulate-location path communicates via standard USB host mode — not via Apple's MFi EAAccessory framework. No Apple authentication chip or program enrollment needed.

Recommended — Before launch

Legal Opinion: CFAA / ECPA

The device is operated by the phone owner on their own device. No unauthorized access to any computer system occurs. A written opinion from outside counsel on CFAA and wire fraud exposure is recommended before commercial launch, particularly regarding the DNS blocking feature.

Required — EU / CA launch

GDPR / CCPA

The backend collects only email address. No real location is ever processed server-side. A Data Protection Impact Assessment (DPIA) is required before EU launch. A DPO appointment may be required if user scale exceeds GDPR thresholds. CCPA compliance is lightweight given minimal PII collection.

Required — App Store

Apple MDM Certificate

The optional DNS privacy feature serves a signed .mobileconfig profile. Apple requires an MDM vendor certificate for signed profiles. Enrollment via Apple Business Manager or third-party MDM vendor program. Required to avoid "unsigned profile" warning in iOS Settings.

Review Risk — Ongoing

App Store Guidelines

App requests no Always location permission; injection occurs entirely on the hardware. The app itself is a control surface — the core session feature is not testable by Apple without physical hardware, which must be declared in review notes. Compliance with Guidelines 5.4 (VPN & DNS) and 2.1 (App Completeness) must be maintained. Legal review of app description framing recommended before first submission. Age rating: 17+.

Pre-launch audit

isSimulatedBySoftware Detection

iOS sets a detectable flag on DVT-injected locations. Empirical testing of Find My, Life360, and Snapchat against this flag is required before launch. A security researcher audit is recommended. This is a product risk, not a legal one, but bears commercial impact.

Known Limitations

Transparent risk register

Medium · Accepted

isSimulatedBySoftware flag

iOS marks DVT-injected locations with CLLocationSourceInformation.isSimulatedBySoftware = true. Any app checking this flag can detect simulation. Current testing of Find My and Life360 shows neither checks in production — but this must be verified empirically per app per update cycle.

Low · Cosmetic

Developer Mode indicator

Developer Mode displays a visible indicator under Settings → Privacy & Security. A motivated inspector examining the phone's settings would notice it. Accepted for v2.0; no mitigation path exists without jailbreak.

Medium · Mitigatable

Wi-Fi / cellular cross-check

Location-tracking apps can independently query Apple/Google network-based location APIs using Wi-Fi scan data. This is a server-side cross-check the device cannot intercept. Mitigated via optional DNS blocking of Apple's location verification domains (apple-dns.net, gsp1.apple.com).

Low · Physical

USB cable requirement

Session requires continuous USB connection. Disconnecting the cable causes iOS to revert to real GPS within 2–5 seconds. ghostd reconnects automatically within 30 seconds if cable is re-inserted. The cable itself is a physical tell that must be managed by the user.
